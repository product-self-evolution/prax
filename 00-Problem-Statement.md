## Problem Framing Narrative

### I am

一个 25-35 岁的互联网**在职从业者**，典型身份：

- 企业内产品经理（Product Manager, PM），面对业务侧 AI 落地压力
- 运营、增长、业务分析（Business Analyst, BA）等需要信息处理与自动化的岗位
- 半技术背景的开发者或技术型产品经理（能看懂代码，不擅长端到端构建系统）

关键特征：

- 有产品认知与业务理解，能识别"这个场景适合 AI"
- 能使用现成的 AI 工具（ChatGPT / Claude / Dify / Coze），但不会独立搭建可运行的 AI 工作流（Workflow）
- 公司有 AI 落地需求，但缺少"企业 sense"的参考实践
- 时间被业务占满，没有整块时间自学 Agent 框架

### Trying to

在真实业务场景中快速落地 AI Agent 能力：

- 把信息聚合、内容生产、数据分析等重复性工作交给 Agent
- 在公司内部推动有说服力的 AI 落地案例
- 自己能动手跑通、给老板演示、给同事复用
- 积累可对外展示的 AI 落地作品（个人品牌资产）

### But

- **教程面向开发者**：市面上的 Agent 教程多基于代码（Python + LangChain），非技术从业者入门成本高
- **玩具级 demo 占多数**：大多数案例是"问个天气"、"查个日程"级别，无法对应企业真实场景
- **缺少场景化模板**：想做"RSS 聚合 + 每日推送 + 企业邮件"这类组合场景时，找不到可 fork 的起点
- **工具选型混乱**：Dify、Coze、n8n、扣子等平台差异不清楚，不敢在公司里推荐
- **无法独立闭环**：学会某个单点能力后，不知道如何串成"可部署、可维护、可复用"的生产级 Agent

### Because

当前 AI Agent 生态分成两极：

- **开发者层**：代码驱动，门槛高，企业 PoC（Proof of Concept, 概念验证）耗时长
- **小白工具层**：无代码，但场景玩具化，无法落地企业级需求

中间层——"**有产品思维、企业场景、可复用、可配置**"的 Agent 模板库——几乎是空的。

### Which makes me feel

- **焦虑**：公司在提 AI 战略，我讲不出具体落地路径
- **挫败**：试过 Dify / Coze，做了 demo 但无法真正服务业务
- **孤立**：没有一个可参考的"企业 PM 的 Agent 作品集"
- **纠结**：想展示自己的 AI 能力，但手里只有零散 demo，没有系统化交付物

## Context & Constraints

- **企业场景优先**：所有模块都必须对应真实企业工作流（如"信息聚合推送"、"竞品追踪"、"客户反馈自动分析"）
- **低代码优先**：所有模块必须可通过 Dify / Coze 这类低代码平台部署，不强制写代码
- **可复用交付物**：每个模块产出 `workflow.yaml` 或等价配置文件，fork 即用
- **部署时间盒**：单模块"从导入到跑通"<= 30 分钟
- **工具链无关性**：设计应兼容主流低代码平台，避免绑死一家
- **开源优先**：MIT License，可被企业内部魔改、再分发
- **中文优先、国际化友好**：面向中文从业者为主，但结构与代码注释支持英文用户 fork

---

## Final Problem Statement

> **有产品思维但不会搭 Agent 的在职从业者**需要一座"企业级 AI Agent 场景模板库"，把典型业务场景变成可 fork 即用的工作流模板，因为现有资源要么面向开发者、要么停留在玩具 demo，导致他们即使看到 AI 的潜力，也无法在公司里跑出能真正服务业务的第一个落地案例。
