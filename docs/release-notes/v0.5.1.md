## TL;DR

v0.5.1 新增两份关键治理文档，让 Prax 的"可测试"和"可贡献"两个维度闭环：

- [`docs/testing/first-run-checklist.md`](docs/testing/first-run-checklist.md) — 让任何人能独立做一次部署测试，帮 Prax 收集 DT (Deploy Time) 和 FDR (First Deploy Rate) 数据
- [`docs/governance/module-quality-checklist.md`](docs/governance/module-quality-checklist.md) — 给贡献者明确的模板质量门槛，含 3 档分级体系和 5 维度评分

这两份文档是 Phase 1 路线图里的必要交付物，本次全部到位。

---

## Why This Matters

v0.5.0 之前 Prax 已经有了模板骨架、架构白皮书、战略文档。但还缺两件事：

1. **如何验证 Prax 真的被跑通了** —— 没有测试脚本，用户"跑不通"的故事就听不到
2. **如何让外部贡献者知道标准** —— 没有质量清单，未来 PR 审核会陷入主观讨论

v0.5.1 把这两个空缺补上。从此：

- 任何人都可以用 `first-run-checklist.md` 做一次标准化测试
- 任何贡献者都可以用 `module-quality-checklist.md` 自检 PR

## New Files

### `docs/testing/first-run-checklist.md`

面向**新用户测试者**的部署测试脚本。核心内容：

- **Pre-Flight Checklist**: 账号、API Key、环境检查
- **Test Protocol**: T0/T1 计时、卡点记录规则（"卡住超过 2 分钟直接记录并跳过"）
- **Feedback Template**: 可直接复制填写，作为 GitHub Issue 提交
- **已知失败模式**: 预先告知已知问题，避免测试者重复反馈

用法：发给 3-5 位朋友做测试，2 周内收集 10+ 份部署反馈。

### `docs/governance/module-quality-checklist.md`

面向**模板贡献者 + PR 审核者**的质量门槛。核心内容：

- **3 档分级体系**: `Experimental` → `Recommended` → `Official`
- **必备交付物**: 6 个必需文件/目录结构
- **README 章节清单**: 11 个必须出现的章节
- **5 维度评分**: Clarity / Runability / Security / Portability / Maintainability（总分 ≥ 20/25 通过）
- **PR 审核流程**: 作者自检 → 维护者 review → 合并后动作 → 升档条件
- **反模式清单**: 9 种直接拒绝情况
- **一键自检命令**: shell 脚本检查必备文件、硬编码凭据、占位符
