# Prax Roadmap

## 1. 路线图总览

本路线图采用三阶段推进，目标是先跑通价值闭环，再逐步增强规模化能力。

```mermaid
flowchart LR
    phase1["Phase1Learn<br/>项目制学习MVP"] --> phase2["Phase2Build<br/>能力资产沉淀"]
    phase2 --> phase3["Phase3Share<br/>开源复用生态"]
```

---

## 2. Phase 1 — Learn（0-8 周）

### 2.1 目标

验证“30 分钟可完成、可运行、可复用”的项目制学习是否能显著提升行动率。

### 2.2 交付范围

1. 建立模块标准模板（`Context -> Goal -> Steps -> Run -> Extend`）  
2. 发布 3-5 个首发模块  
3. 提供一键开始说明与依赖安装指引  
4. 输出统一的模块质量检查清单（可执行、可复现、可扩展）

### 2.3 首发模块建议

- `ai-digest`：AI 周报聚合与摘要生成
- `content-repurpose`：单篇内容多平台改写
- `03-competitor-bot`：竞品追踪与摘要机器人
- `04-local-rag`：本地知识问答 (Retrieval-Augmented Generation, 检索增强生成)
- `workflow-starter`：个人工作流编排起步模板

### 2.4 验证指标

- 新用户首模块完成率 >= 50%
- 首次价值达成时间（TTV）中位数 <= 30 分钟
- 模块运行成功率 >= 80%

### 2.5 主要风险与缓解

- **风险**：模块步骤过长导致中途流失  
  **缓解**：强制拆分为“必做步骤（15 分钟）+ 扩展步骤（15 分钟）”
- **风险**：环境依赖导致启动失败  
  **缓解**：提供环境检测脚本与最小依赖版本清单

---

## 3. Phase 2 — Build（9-16 周）

### 3.1 目标

让用户把“做出来的项目”自动沉淀为“看得见的能力资产”。

### 3.2 交付范围

1. 作品集页面生成器（从模块产出自动生成展示页）
2. 个人能力画像（按模块标签汇总能力维度）
3. 标准化展示模版（项目背景、目标、方案、结果、可复用链接）

### 3.3 验证指标

- 作品页生成率 >= 40%
- 生成作品页后二次编辑率 >= 30%（说明有展示意愿）
- 单用户平均可展示项目数 >= 2

### 3.4 主要风险与缓解

- **风险**：用户产出质量参差，作品页价值不足  
  **缓解**：提供最小展示标准与自动补全引导
- **风险**：模板同质化严重  
  **缓解**：支持多种展示模板与自定义区块

---

## 4. Phase 3 — Share（17-24 周）

### 4.1 目标

形成“发布 -> fork -> 改造 -> 回流贡献”的开源增长飞轮。

### 4.2 交付范围

1. 工作流发布规范（metadata、依赖、输入输出、许可）
2. Fork 指南与改造指南
3. 工作流索引页（按场景、难度、工具分类）
4. 贡献者流程（Issue 模板、PR 模板、评审标准）

### 4.3 验证指标

- 每个工作流平均 fork 数 >= 3
- 社区贡献 PR（Pull Request, 合并请求）月增稳定
- 二次改造工作流占比 >= 25%

### 4.4 主要风险与缓解

- **风险**：冷启动导致内容稀缺  
  **缓解**：先由官方发布 10+ 高质量样板工作流
- **风险**：质量失控影响信任  
  **缓解**：建立分级审核（实验/推荐/官方）标签体系

---

## 5. 横向工作流（跨阶段持续）

### 5.1 文档体系

- `README.md`：对外叙事与快速开始
- `docs/vision.md`：战略方向与边界
- `docs/roadmap.md`：执行计划与里程碑
- `CONTRIBUTING.md`：贡献流程规范（Phase 3 前完成）

### 5.2 数据观测

建立最小观测面板，追踪：

- 模块开始数、完成数、放弃点
- 作品页生成行为
- 发布与 fork 行为

### 5.3 品牌内容联动

每个模块都对应一条公开内容资产：

- 模块发布贴（你做了什么）
- 复盘贴（为什么这样做）
- 模板贴（别人如何复用）

---

## 6. 发布节奏建议（双周迭代）

采用双周迭代（Sprint, 冲刺）：

- 第 1 周：需求定义 + 开发 + 文档草稿
- 第 2 周：验证 + 修复 + 发布 + 复盘

每个 Sprint 固定输出：

1. 一个可运行增量
2. 一篇公开说明
3. 一份复盘记录（保留决策依据）

---

## 7. 下一步执行清单（马上可做）

1. 确认 Phase 1 的 3 个必做模块（从 5 个候选中先选 3）
2. 为每个模块补齐统一模板（输入、输出、步骤、扩展）
3. 建立 `modules/` 与 `templates/` 的基础目录
4. 完成第一个模块端到端发布（含文档与示例输出）
5. 用真实用户（3-5 人）做首轮可用性访谈与完成率验证
